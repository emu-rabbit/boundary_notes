# 既有產品體驗契約

## 文件角色

本文件保存 Boundary Notes 已確認、跨多次任務仍需維持的 feature-level UI 與互動契約。跨功能設計原則由 `.agents/skills/professional/ui_ux_standards.md` 管理；資料與回答語意由 `.agents/specs/question_bank_and_secret_file_system.md` 管理。

這些契約預設屬於 **Default**；涉及知情同意、敏感資料、法律文件或分享揭露的部分屬於 **Invariant**。若新證據顯示既有構圖不再服務使用者目標，可以提出可逆替代方案，但不得默默移除 invariant。

## 前導與建立檔案劇情

- 四語公開入口直接沿用既有前導，不插入 SEO 專用 landing。第一次進入由語系入口開始前導；已有 profile name 的回訪者由相同入口直接前往同語系 Home。
- `/{locale}/intro` 保留為顯式重播入口，不套用回訪攔截；已播放過的前導提供低視覺重量的底線「跳過」連結。
- 前導劇情維持單頁 RPG 對話畫面感，以固定場景、角色與對話區切換狀態；手機版不退化成傳統長頁閱讀。
- 建立新檔案的劇情序幕沿用前導劇情的場景、角色與緊湊對話卡，不改成置中資訊頁或拉長對話卡。
- 前導初次相遇的說話者稱為「不知哪來的兔子」；只有進入建立測驗的劇情後才稱為「守密兔」。
- 劇情分支內部標記不得出現在正式文案；非主要操作的小型輔助文字置中，不干擾主要選項。
- 建立新檔案的重播劇情必須先選擇問題範圍才能跳過；時光機必須先選擇兩份檔案才能略過敘事進入比對。

## 高重複作答流程

- 建立檔案先選擇主導側、配合側或皆顯示，再告知分類與細項數量，先完成 `其他` 以外的分類層，之後由結果頁自由進入細項。
- 劇情導覽和逐題作答不提供回 Home 的主要入口，也不額外提供「暫停／稍後繼續」按鈕；目前題次與即時保存必須清楚。
- 從結果頁進入分類時，入口只依目前方向的細項進度判斷：全未答或全答直接顯示全部；部分作答才詢問「僅顯示未作答」或「顯示全部」。一般分類的全部模式包含分類題與細項，`其他` 只有細項。
- 分類與細項作答可能重複數十至數百次，應優先讓核心操作在單頁內可見。
- 使用者必須清楚知道目前位置，並能返回上一題；不為了顯示百分比額外增加進度條。
- 每題保留分類視覺錨點，避免重複顯示同一方向語意；必填選項使用容易辨識的圖示與短標籤。
- 兩項必填答案完成後立即保存。保留明確的「下一題」按鈕，也可以在短暫確認後自動前往下一題。
- 分類題顯示方向性標題、描述與細項預覽入口；細項題顯示方向正確的 `title`，description 相同時不重複。warning 使用固定前綴並保留題庫原文。

## 編輯結果頁

- 編輯與唯讀使用不同資訊架構。編輯頁優先顯示填答狀態與分類細項入口；唯讀頁優先服務分享與閱讀。
- 兩者都使用檔案內 `profileName` 組成標題，不用「你的」取代擁有者名稱。
- 編輯頁提供回 Home、另開新分頁預覽與建立雲端分享；不顯示唯讀頁的溝通警語。
- 分類卡包含分類圖片、分類層答案、備註摘要與細項完成度，整張卡片可進入編輯。經驗與喜好合成自然一句話，不退化成兩欄報表。
- 進度使用目前方向、scope 內已回答細項數除以 runtime 適用細項總數，被篩除題目不進分母。
- 經驗與喜好都為 `seeDetails` 時只顯示「參考詳細」；只有一欄時明示是哪一欄。
- `其他` 固定最後，只顯示細項完成度與入口，不偽造分類層答案。
- `all` scope 使用主導側／配合側切換器，不在同一卡片並列兩個方向；手機切換器位於焦點喜好與分類之前。
- 離開分類細項時保留剛才方向，並在 route scroll reset 後回到該分類附近。返回使用清楚的上層箭頭，不使用文件 icon。

### Spotlight 編輯

- 主導側與配合側各自有五個固定容量，候選與儲存池不得共用。
- 候選只包含目前方向中已回答且喜好為 `like` 或 `love` 的分類或細項；以分類分組，每組先 `love` 再 `like`。
- 使用者只能點擊已登記格或第一個空格；刪除後同方向後方項目依序前移。
- 目前方向已由頁面切換器表達，卡片、目前選擇與候選不重複方向尾標。
- 分類、標題與說明完整顯示，不使用省略號或在窄版隱藏說明。Spotlight 是手動整理，不呈現為自動推薦、社交排行或鼓勵清單。

## 時光機

### 階段與摘要

- 前導劇情結束後直接進入獨立的兩檔差異工作區，不重播結語，也不沿用故事對話構圖。
- 比對方向固定清楚表達較早到較新，並交代新檔名稱、必要的舊暱稱、兩份最後編輯時間與 scope。
- 手機首屏將返回與重選放在同列；其他身份與時間資訊收進一張無巢狀卡片的檔案摘要。日期維持較早到較新的左右對照，方向切換是摘要最後一列。
- 選檔清單每列只保留名稱、必要來源、scope 與日期。同一來源使用群組表達，不逐列重複；重新開啟 dialog 時回到目前選項或清單頂端，不露出被裁切的半張選項。

### 差異區塊

只顯示有變動的區塊，順序固定為：

1. 焦點喜好
2. 大分類
3. 絕對禁止底線
4. 其他細項

焦點喜好、大分類與絕對禁止各顯示前 4 筆預覽；其他細項因採密集列表可顯示前 8 筆。完整內容由單一 dialog 承接。

- **焦點喜好**：使用唯讀檔案的克制暖金語彙作為第一視覺主角。排序依序為成為第 1 位、其他新加入、排行向前、排行向後、被移除；新加入且直接成為第 1 位歸入第一類。暖金只用於區塊標線、一般字重且無陰影的標題與第一筆結果，其餘內容回到暖白／玫瑰色。
- **大分類**：使用完整且平衡的分類卡。同列卡片內容較少時仍從上緣開始，不使用垂直置中。左側分類名稱是唯一標題，右側不重複「分類整體」；只有無法取得歷史分類名稱時使用 fallback。完整 dialog 以上方小圖加分類名建立辨識，下一列狀態面使用完整寬度。單張或奇數末張仍沿用單一中央方向與較早／較新整組狀態面，未變欄位省略；只有一個主要欄位時由該欄位撐滿。備註放在狀態面底部、分隔線之後；狀態面只有備註時移除無意義的分隔線與間距。
- **絕對禁止**：使用克制的莓紅底與實體邊界。固定保留「新列為絕對禁止」與「已移出絕對禁止」兩欄，每筆只顯示分類圖、項目名與進入或離開 `hardNo` 的方向；不在此區重複經驗或備註。空欄以低視覺重量的「無」保留閱讀座標。兩欄各自獨立套用前 4 筆預覽上限；只有任一欄本身超過上限時才顯示完整入口。
- **其他細項**：使用滿寬、較小圖面、低陰影與緊密分隔線的核對節奏。每題只保留一個中央箭頭，把經驗、喜好與備註分別收進較早與較新的整組狀態面。

分類圖是一個分類群組的主要辨識面：使用一張足以辨識的 1:1 圖搭配一次分類名，統領該分類所有變動；不在逐筆內容重複圖片與分類名。完整 dialog 沿用各區自身構圖與分類分組。

題目只有在兩檔皆已作答時才列入比較。scope 新增或移除造成整側沒有答案時，只顯示一次先前／之後未有作答紀錄，不展開成逐題差異。不得加入總變動數、檔案 ID、版本標籤、重複方向文字、逐區無變動狀態或框中再包框的報表層次；四區皆空且沒有整側提示時才顯示一次整體空狀態。

## 唯讀秘密檔案

### 總覽與警語

- 唯讀頁是獨立 route、view 與 component，不以編輯頁內大量 `mode` 分支實作；視覺可沿用編輯頁語彙，但有自己的內容、功能與 CSS owner。
- 不提供修改答案或建立雲端檔案。從編輯頁進入時使用瀏覽器原生 `_blank` 連結開啟完整 preview route。
- 必要警語為「這份測驗結果僅供參考，並無法完整的描述XXX的喜好或特質，也請勿用來替代任何必要的溝通。」其中 `XXX` 使用檔案 `profileName`，缺少名稱時使用「兔子」。
- 總覽不顯示百分比或已答題數；分類仍完整列出，未作答分類標示尚未填寫。`其他` 不標成分類未作答，說明結果從細項開始。
- 分類層答案可使用一句自然摘要；分類詳情的每個細項使用經驗／喜好兩欄，方便逐項核對。
- 分類層喜好為 `love` 的卡片可使用暖金色底；`其他` 維持一般深色卡片。
- 焦點喜好最多五項直接顯示，沒有內容時整區隱藏；使用名次層級與尺寸差異，但避免過度高瘦。
- JSON 匯入後預設先開唯讀模式；舊檔案列表將「檢視結果」與「編輯結果」分成不同入口。
- 人物指稱使用檔案內 `profileName`，不假設閱讀者就是檔案擁有者。

### 絕對禁止入口

- 分類層 `hardNo` 在總覽如實呈現；細項 `hardNo` 不逐筆攤在總覽，而以分類相簿標題附近、帶數量的入口開啟 dialog。
- 入口與 dialog 只計算目前方向，不跨方向合併，也不在小標重複方向尾標；入口不放在身份摘要側欄。
- 目前方向為零項時不渲染入口、dialog 或空白占位。分類詳情仍如實顯示細項答案。
- `hardNo` 不觸發額外保護流程；`reluctant` 命名保留。

### 分類圖片與詳情

- 結果頁的 1:1 分類圖保留主要物件完整可辨識；總覽、焦點喜好與分類詳情在桌機和手機都使用 `contain` 或等價構圖，不因備註撐高卡片而同步拉高並裁切圖片。
- 進入唯讀分類詳情後，隱藏總覽用的檔案身份、警語、語系與操作側欄，只保留返回、分類摘要與細項答案。
- 桌機使用左摘要、右返回與列表；手機依返回、摘要、列表排列。不增加「細項結果／分類細項」等重複標題。
- 詳情列出所有適用細項；已回答項目顯示經驗、喜好、備註與警示，未回答標示「未作答」，`filteredOut` 不顯示。
- 返回總覽時保留方向與原本捲動位置，不增加上一分類／下一分類。

### 語系與操作

- 語系切換只存在於唯讀總覽內容欄上方的低視覺重量工具列，不放進摘要側欄，也不與焦點喜好或分類標題競爭。
- 控制項平時使用弱邊界與次要文字色，hover／focus 時提高辨識度，並保留明確文字標籤。
- 「建立我的檔案」只在 `source=cloud` 時顯示，是雲端觀看者的主要 CTA；手機版固定於 viewport 底部並預留安全空間。本地唯讀預覽不顯示。
- 分享操作放在閱覽語言工具附近，使用具可及名稱的小型 icon；不放在檔案摘要區。贊助入口維持摘要區的低視覺重量。

### 焦點喜好入口

- 只有唯讀檢視的焦點喜好區塊可作為整塊點擊面。
- 目前方向存在「很喜歡」項目時，開啟沿用絕對禁止清單構圖的 dialog，集中列出該方向所有標示為「很喜歡」的分類與細項。
- 區塊右下角只保留低視覺重量的「查看 N 個很喜歡的項目 →」，引導點擊整個區塊。
- 焦點區維持深色、克制層次，不使用大面積亮金色；編輯結果頁不新增此入口或 dialog。

## 個人化標題

- 「{暱稱}的祕密檔案」由共用元件拆成暱稱、語系連接詞與固定產品名片段。
- 名稱長度以視覺寬度量測並決定縮字密度，不以單純字數一刀切，也不靜默裁切使用者名稱。
- 畫面優先使用雙行 title，暱稱保留在大標、連接詞字級較小、固定產品名不可拆字；在關鍵寬度依內容縮小，不放任不自然斷句。

## 分享圖片

- 分享圖沿用網頁 locale font tokens：繁體中文與英文優先 Huninn／粉圓體、簡體中文優先 Noto Sans SC、日文優先 Noto Sans JP，並保留系統 CJK fallback。
- 量測前等待本次輸出字元的字型載入。SVG 轉 canvas 後用輸出範圍外文字探針確認 glyph 像素；首次仍在載入時以新 SVG 重畫。持續無字時中止，不建立空白文字下載。
- 粉圓體不大量使用 700 合成粗體。優先用字級、色彩與區塊層級建立重點；描述維持一般字重，卡片重點與區塊標題只克制加重。

## 一般內容頁

- Home 是多個獨立功能的入口，不是永久 landing 或單一流程。建立檔案、查看舊檔案與時光機使用主要入口；關於、設定與贊助使用較低視覺重量的文字入口，不以同等大按鈕或右上角 icon 破壞重心。
- Home 與秘密檔案檢視頁的贊助入口開啟同一個 sponsor dialog。
- 主頁、設定、關於與其他功能頁可依內容選擇單頁或自然文件捲動；route 切換從頁面頂端開始，背景不因捲動位置放大、拉伸或跳動。
- 關於頁不是長型 landing page。桌機優先在接近一屏內完成主要內容與必要入口；使命、角色與產品界線合併成一次清楚說明，不展示內部防護實作。保留敏感內容提醒、GitHub Issue、電子郵件、使用條款、隱私權政策與版本歷史入口。重播前導可疊放兔子插畫，返回入口放在頁面上方角落。
- 版本歷史從頁面頂端開始，標題區只保留 Boundary Notes 品牌小標與頁面標題。目前版本使用高對比醒目色，只有較舊版本淡化。
- 使用條款與隱私權政策從產品頁以新分頁開啟，使用單頁靜態閱讀版面且不放返回鍵；保留完整四語切換，以繁體中文版本為準。法律頁不得顯示分析同意 banner。
- 產品維持單一暗色主題，不建立明亮／暗黑模式切換。
