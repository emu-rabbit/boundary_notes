# 任務結束文件同步規範技能

## 概述

本技能要求 Agent 在每次任務結束前，自動巡查與本次變更相關的文件，並在必要時更新至最新狀態。目標是避免文件殘留舊狀態、舊決策、失效流程或與實際專案不一致的描述。

## 觸發條件

- 每次任務即將結束、準備回覆使用者前。
- 變更程式碼、流程、產品行為、架構、設定、文件或專案規範後。
- 使用者提出重要決策、新限制、新工作流程或新的長期規範時。

## 核心規則

- **每次任務都要執行**：任務完成前，Agent 必須主動巡查與本次變更相關的文件。
- **文件跟上現況**：若程式碼、流程、架構、決策、限制、使用方式或專案規範已改變，相關文件必須同步更新。
- **清除舊狀態**：不可留下已過期的決策、錯誤流程、失效指令或與現況不符的說法。
- **範圍精準**：只更新與本次任務相關的文件；若發現大型文件債或不確定是否應改動，先回報並詢問使用者。
- **無需更新也要確認**：若巡查後判斷沒有相關文件需要更新，應在收尾時簡短說明已檢查且不需更新。
- **決策與意圖同步**：若本次任務新增或修改重要決策、目標約束、非目標或整體意圖，必須依 `.agents/skills/core/decision_traceability.md` 處理。

## 執行步驟

1. **辨識影響範圍**：判斷本次任務影響哪些程式碼、流程、產品行為、規範或決策。
2. **搜尋相關文件**：檢查 README、AGENTS、skills、decisions、architecture、mission、規格文件或註解。
3. **比對現況**：確認文件是否仍反映最新程式碼、決策、限制與使用方式。
4. **同步更新**：若文件過期，直接更新相關段落，避免大範圍無關重寫。
5. **收尾說明**：在最終回覆中簡短說明已更新哪些文件，或說明已檢查且不需更新。

## 注意事項

- 本技能是任務收尾流程，不負責決定實作方向。
- 文件同步應維持精準、低噪音。
- 若文件更新牽涉尚未確認的產品或架構決策，必須先詢問使用者。
