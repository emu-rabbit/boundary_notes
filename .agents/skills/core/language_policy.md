# 繁體中文溝通與文件規範技能

## 概述

本技能確保 Agent 在與使用者互動、撰寫計畫、記錄進度以及產出任何專案文件時，皆保持一致的繁體中文語系，以符合使用者偏好。

## 觸發條件

- 任何時候 Agent 需要對使用者做出回應。
- 建立或更新 `.agents`、README、設計文件、任務文件或專案說明。
- 撰寫程式碼註解，除非該程式碼已有既定語言慣例。

## 核心規則

- **語系優先**：所有使用者可見輸出與專案文件必須以繁體中文為主。
- **保留詞彙**：技術關鍵字、API、命令、程式識別字、套件名稱與無法精準翻譯的專有名詞可保留英文。
- **一致性**：同一概念在不同文件中的中文譯名應保持一致。
- **自然語氣**：文件與回覆應自然、清楚、可直接使用，避免僵硬翻譯腔。

## 執行步驟

1. 產出內容前，確認是否混入簡體中文或不必要的英文敘述。
2. 若發現語系不一致，改寫為自然繁體中文。
3. 對 code、CSS class、檔案路徑、命令與 API 保留原文，避免造成技術含義偏移。

## 注意事項

- 本技能具有高優先級，除非使用者明確要求其他語言。
- 若內容是給國際讀者看的外部文案，可依使用者指定建立英文版，但 Agent 與專案規範仍預設繁體中文。
