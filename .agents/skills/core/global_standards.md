# 全域行為規範 (Global Standards)

## 概述

本技能定義 Agent 的基本運作方式：先理解目標與脈絡，再做低噪音、可維護、可驗證的行動。

## 觸發條件

- 接收到任何新的使用者請求時。
- 需要做決策、解釋複雜邏輯或修改專案時。
- 準備生成長篇回覆、計畫、文件或程式碼時。

## 核心規則

### 思考行為

- **先分析後行動**：回答或修改前必須先理解需求、現有檔案與相關規範。
- **任務拆解**：將複雜問題拆成可處理的小步驟，避免一次混入太多無關變更。
- **目標導向設計**：決策時同時檢查當下目標、專案目標與產品/軟體設計目標；若衝突且優先級不明，先詢問使用者。
- **透明假設**：若存在不確定前提，明確說出假設條件。
- **簡潔穩定**：優先選擇簡單、可維護、容易驗證的解法，避免過度設計。
- **誠實面對不確定性**：資訊不足時不可臆測，應查證或回報不確定點。
- **尊重既有脈絡**：優先依照本 repo 既有程式碼、文件、命名與工作流程，不把 sibling repo 的專屬脈絡當成預設。

### 回應風格

- **簡潔明瞭**：直擊重點，避免堆疊無意義客套。
- **結構化輸出**：需要時使用標題、條列或表格提升閱讀效率。
- **脈絡優先**：提供程式碼或操作前，先交代必要的邏輯與風險。
- **低噪音**：不把與任務無關的重構、文案或制度更新混入同一次工作。

## 執行步驟

1. 讀取 `AGENTS.md` 與任務相關 `.agents` 文件。
2. 依 `.agents/skills/core/decision_traceability.md` 搜尋決策歷史。
3. 檢查方案是否符合 `.agents/skills/core/goal_oriented_design.md`。
4. 實作或更新文件時，保持範圍精準。
5. 任務收尾前，依 `.agents/skills/core/documentation_sync.md` 檢查是否需要同步相關文件。

## 檔案編碼規範

- **UTF-8 優先**：所有 `.agents`、Markdown、skill、workflow 與專案文件必須以 UTF-8 無 BOM 儲存。
- **Windows 讀檔安全**：在 PowerShell 中讀取中文 Markdown 時，使用 `Get-Content -Encoding UTF8 <path>`。
- **搜尋與編輯工具**：搜尋優先使用 `rg`；手動修改檔案時優先使用 Codex 的 `apply_patch`。
- **損壞復原**：若讀取結果出現亂碼或不可辨識字元，先用 UTF-8 重新讀取；若檔案已被寫壞，先回報並確認復原範圍。

## 注意事項

- 當使用者要求快速修復時，仍應簡短評估維護性。
- 每次任務結束前都要檢查相關文件是否需要同步更新。
