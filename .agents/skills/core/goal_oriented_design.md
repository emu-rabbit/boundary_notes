# 目標導向設計規範技能

## 概述

本技能要求 Agent 在拆解任務、規劃做法與做出決策時，以目標導向設計為核心。Agent 不只要完成使用者當下的小目標，也必須檢查做法是否符合本專案目前已確認的方向與長期維護性。

## 觸發條件

- 拆解任務、擬定計畫或選擇實作方向時。
- 需要在多個方案之間做取捨時。
- 變更程式碼、設計、流程、架構、文件或專案規範時。

## 核心規則

- **當下目標**：先確認使用者此刻明確提出的需求、限制與交付成果。
- **專案目標**：檢查做法是否符合 `AGENTS.md`、既有 `.agents` 文件、README、規格文件與使用者最新指示。
- **使命待定時不擅自補完**：若專案使命、產品定位或技術方向尚未建立，不要用 sibling repo 的使命替代；以低噪音、可回退、易維護的做法前進。
- **設計目標**：從軟體設計、產品設計、視覺設計、可用性與維護性檢查方案是否合理。
- **過往決策與相關文件**：做決策前，依 `.agents/skills/core/decision_traceability.md` 搜尋 `.agents/decisions/decision_history.md`，並讀取任務相關文件。
- **任務中目標可沉澱**：若使用者確認新的長期目標、限制或非目標，必須同步到最適合的既有文件；若需要新文件，先詢問使用者。
- **避免局部最佳化**：不可只解決局部問題，卻破壞整體一致性、效能、可維護性或專案定位。
- **衝突時停止詢問**：若當下目標、既有專案目標與設計目標衝突且優先級不明，必須先詢問使用者。

## 執行步驟

1. **建立目標階層**：列出當下目標、既有決策、任務相關文件中的有效約束、已確認的專案目標與設計目標。
2. **檢查一致性**：確認方案是否同時滿足這些目標與約束。
3. **處理衝突**：若優先級不明，停止執行並向使用者詢問。
4. **執行任務**：在不偏離整體目標的前提下完成實作或文件變更。

## 注意事項

- 本技能不是要求每次都寫長篇計畫，而是要求每次決策都能對齊目標。
- 若專案尚未建立完整程式碼或使命文件，應以 `AGENTS.md`、已存在文件與使用者最新指示為主要依據。
