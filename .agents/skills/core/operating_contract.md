# Agent 操作契約

## 文件角色

本文件是所有 Boundary Notes 任務的共通操作基準。它只規範 Agent 如何理解、執行與交付任務；產品、架構、UI、領域與功能真相由 `AGENTS.md` 路由到各自的 canonical owner。

## 基本契約

- **先確認結果**：先理解使用者要得到的結果、明示限制、非目標與可接受證據，再選擇做法。
- **查實際 owner**：依 `AGENTS.md` 讀取任務相關文件，並檢查現有 code、config、tests、built output 或 live surface；文件記憶不是 current-state 證據。
- **搜尋決策脈絡**：用任務關鍵字搜尋 `.agents/decisions/decision_history.md`。只有命中的相關決策需要完整閱讀。
- **維持範圍**：不把無關重構、文案、制度、部署或其他 Agent 的變更混入任務。
- **透明假設**：可安全推進的假設應明說；會改變產品方向、資料邊界、外部狀態或授權範圍的假設必須先詢問。
- **尊重現有工作樹**：現有 modified、staged、untracked 內容預設屬於使用者或其他工作，不得擅自刪除、覆寫或提交。
- **簡單且可回退**：優先沿用現有 owner、資料流與元件；只有真實重複或複雜度需要時才增加抽象。
- **證據對應風險**：文件使用 diff 與引用檢查；程式邏輯使用 tests/typecheck；build、UI、資料或 deployment 依風險增加相應驗證。
- **分類工具失敗**：區分 sandbox、權限、網路、平台與程式錯誤。環境失敗不是修改產品邏輯的理由。
- **誠實交付**：沒有完成的驗證、外部阻塞、殘留風險與未處理工作必須清楚回報。

## 語言與溝通

- 使用者可見回覆與 repository 文件預設使用自然繁體中文。
- 技術關鍵字、commands、API、套件、code identifiers 與既定名稱保留原文。
- 回覆先交代結果與重要判斷；只使用能提升理解的結構，不輸出無意義的程序流水帳。
- 使用者提供的品牌、標題、文案與互動意圖應精準保留，不用泛化措辭取代。

## 任務流程

1. 讀取 `AGENTS.md` 與任務相關 canonical 文件。
2. 搜尋相關 decision 與實際 code/config owner。
3. 檢查工作樹與變更邊界。
4. 以最小、完整、可驗證的範圍執行。
5. 執行與風險相稱的驗證。
6. 若產生持久文件變更，依 `.agents/skills/core/documentation_governance.md` 維護文件庫。
7. 回報結果、驗證、殘留工作與未觸碰的外部變更。

## 允許挑戰既有預設

Default 與 snapshot 不是禁止思考的命令。若實際程式、使用者目標或新證據顯示現有做法已不再合適，Agent 可以：

1. 指出被挑戰的規則層級與 canonical owner。
2. 說明目前做法造成的具體成本。
3. 提出可逆替代方案與驗證方式。
4. 在不越過 invariant、decision 或使用者授權的前提下實作；否則先取得確認。

不得用「遵守文件」取代專業判斷，也不得用「創新」繞過安全與資料邊界。
