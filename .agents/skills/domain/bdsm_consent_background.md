# BDSM 與知情同意背景技能

## 概述

本技能是本專案正式領域規範，提供後續 Agent 在處理本專案 BDSM（愉虐）題庫、文案、UI、風險警示與分享流程時必須遵守的背景與產品邊界。它不是 BDSM 實踐教學，也不是鼓勵任何行為的指南；它的用途是讓產品能以自我理解、界限核對與知情同意為中心。內容可以具有教育性質，但產品不以教育為主要目的，也不得以教育工具、教育服務或教育者身分對外宣稱。

## 觸發條件

- 建立或修改 BDSM 互動項目、題庫分類、強度/時間/角色方向資料。
- 撰寫測驗說明、結果頁、分享頁、警示文案、同意提醒或安全提示。
- 設計秘密檔案、分享連結、歷史檔案、時間軸或檢閱他人檔案的流程。
- 評估特定項目是否需要高風險、法律風險、醫療風險或第三方同意警示。

## 基本背景

BDSM 通常被理解為涵蓋 Bondage/Discipline、Dominance/Submission、Sadism/Masochism 等範圍的總稱。不同社群、語言與文化對詞彙會有差異；本專案不應試圖替使用者定義身份，而應專注於具體互動項目的理解與溝通。

在 BDSM 脈絡中，「同意」不是一次性的勾選，而是建立在資訊、能力、自由意願、界限、可撤回性與持續溝通上的過程。常見背景概念包含：

- `SSC`：Safe, Sane and Consensual，強調安全、理智與同意。
- `RACK`：Risk-Aware Consensual Kink，強調參與者理解風險後的知情同意。
- `limits`：界限，可包含不可接受、需要條件、需要慢慢探索或目前不確定的項目。
- `safeword` / 安全詞：用來降低強度、暫停或停止的明確訊號；若語言無法使用，也可能需要非語言訊號。
- negotiation / 事前協商：在任何互動前討論項目、界限、風險、停止方式與照護需求。

本專案應借用這些概念作為「界限核對筆記本」的語意背景，但不應把它們包裝成實踐教學。

## 產品內的語意規則

題目與結果設計必須避免把不同語意混在一起：

- `經驗多寡` 只表示使用者自認有多少經驗，不表示喜歡、不表示接受，也不表示能力足夠。
- `感興趣程度` 只表示目前興趣，不表示同意、不表示承諾，也不表示想立刻實踐。
- `備註` 可能包含界限、條件、顧慮、創傷觸發、偏好、提醒或拒絕，應被視為高優先級語意。
- 空白、不確定、低分、矛盾或保留態度都應被尊重，不應被 UI 自動轉成可接受。
- 檔案只代表填寫當下的自我描述；界限與興趣可能隨時間、對象、身體狀態、心理狀態與關係脈絡改變。

## 題庫設計原則

題庫應以「互動項目」為核心，而非人格屬性或身份標籤。

本專案正式題庫來源、分類層/細項層結構、Google Sheet 匯入規則、回答狀態與秘密檔案資料格式，定義於 `.agents/specs/question_bank_and_secret_file_system.md`。處理題庫或翻譯時，必須同時讀取該文件。

建立題目時應盡量把下列維度拆清楚：

- 互動方向：施予、接受、觀看、被觀看、共同參與等。
- 強度：輕度、中度、重度或其他明確分級。
- 時間長度：短暫、持續、反覆、長時間等。
- 身體部位或心理主題：只在必要時描述，避免露骨與刺激性文案。
- 情境條件：私密、半公開、公共、工具使用、限制行動、身體風險、第三方可見等。
- 風險等級：一般、需提醒、高風險、可能觸法或不適合由本產品詳述。

同一大類項目若角色方向、強度或情境不同，應拆成不同題目。例如「施予臀部拍打-重度」與「接受臀部拍打-輕度」不可合併成單一題目。

## 高風險與警示原則

對於可能造成身體傷害、醫療風險、心理風險、法律風險或第三方未同意暴露的項目，產品應提供清楚警示。警示目標是讓使用者辨識風險並回到現實溝通，不是提供實作方法。

應特別謹慎的項目包含但不限於：

- 窒息、勒頸、限制呼吸或意識改變相關內容。
- 見血、穿刺、切割、感染風險或任何破皮行為。
- 束縛、限制行動、長時間姿勢固定、疼痛或可能留下傷痕的行為。
- 室外露出、公共場所、半公共場所或可能讓未同意第三方看見、聽見、被拍攝或被捲入的行為。
- 酒精、藥物、疲勞、情緒危機或其他會影響判斷與撤回同意能力的狀態。

警示文案應避免：

- 教使用者如何操作高風險行為。
- 暗示只要有同意就一定合法或安全。
- 提供規避法律、規避平台規範、躲避他人發現的建議。
- 把高風險項目寫得刺激、浪漫化或像挑戰清單。

若 Google Sheet 的 `警示` 欄已有內容，UI 應以 `高風險項目：{警示原文}` 顯示，不得擅自改寫成操作教學。若 Agent 認為警示不足，需要先查證可靠來源並保守補強，不得只憑常識新增醫療或法律結論。

## 分享與檢閱他人檔案

秘密檔案分享功能必須把敏感性與同意語意放在核心：

- 分享連結是溝通工具，不是同意書、契約、授權書或承諾。
- 檢閱他人檔案時，應提醒觀看者仍需當面或另行確認當下意願、界限、狀態與停止方式。
- 檔案內容不應被設計成對他人施壓的證據，例如「你之前填過有興趣」不等於現在同意。
- 使用者對現實互動的同意永遠可以撤回，但這不代表已建立的雲端檔案可撤回。第一版雲端檔案長期不可編輯、不可由一般使用者刪除或撤回且永不過期，僅能因法律要求由網站管理員移除；分享流程必須在建立前清楚揭露這項資料保存邊界。

## 語氣與呈現

使用者可見內容應溫柔、清楚、可陪伴，不應冷硬、獵奇或刺激慾望。可使用「目前」、「可以先記下」、「不確定也可以」、「這只是溝通起點」等語氣，降低作答壓力。

插畫兔子可以承擔陪伴、提醒、引導、確認與轉場角色，但不得呈現露骨、NSFW 或煽情畫面。涉及敏感項目時，兔子應以中性道具、表情、便條、資料夾、鎖頭、警示牌等隱喻表達。生成或延伸主角兔子圖片時，必須同時遵守 `.agents/skills/visual/main_rabbit_image_generation.md`，並參考 `.agents/assets/characters/main_rabbit_role.png`。
