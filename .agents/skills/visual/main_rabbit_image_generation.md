# 主角兔子人物設計與生圖規範技能

## 概述

本技能定義「XX的祕密檔案」主角兔子的正式人物設計核心，供後續 Agent 在生成、修改、外包描述、整理或評估任何主角兔子圖片時使用。

核心角色設定檔：

1. `.agents/assets/characters/main_rabbit_role.png`

該圖片是目前主角兔子的最高優先級視覺 reference。未來所有主角兔子相關生圖、插畫延伸、姿勢設計、表情設計、UI mascot asset 或宣傳素材，都必須先參考此檔案，不得只依文字印象自由發揮。

## 觸發條件

- 生成新的主角兔子圖片、表情、姿勢、情境圖、UI 插畫或宣傳素材。
- 修改、重繪、裁切、延伸或評估既有主角兔子圖片。
- 撰寫給影像模型、插畫師或其他 Agent 的主角兔子 prompt / brief。
- 建立與主角兔子相關的 asset naming、preload、style guide 或 UI 使用規則。

## 核心角色設定

主角是一隻純白色、柔和、安靜、有陪伴感的插畫兔子。角色定位不是性感角色、BDSM 實踐者或使用者分類象徵，而是陪伴使用者整理秘密檔案、核對情緒與守護界線的溫柔引導者。

視覺特徵應以 `.agents/assets/characters/main_rabbit_role.png` 為準：

- 身體與頭部以圓潤、簡化、柔軟的幾何感呈現。
- 主色為純白色，參考色碼 `#FFFFFF`。
- 內耳與腮紅為極淡粉色，參考色碼 `#FFECEC`。
- 線條為柔和深灰/暖黑，不使用硬銳黑線或高對比漫畫線。
- 眼睛小而簡潔，表情克制、安靜、親近。
- 嘴部固定使用小型橫向拉鍊嘴，不使用 X 形嘴。拉鍊嘴代表守信、保密與隱私，象徵兔子會好好收著使用者的秘密，除非使用者親自按下「分享」。
- 拉鍊嘴的長度比例必須接近核心設定圖：短而克制，約只佔臉部寬度的一小段，不可拉長成橫跨臉中央的大拉鍊，也不可因表情或姿勢變成明顯微笑線、厚重金屬條或過度寫實的嘴部裝飾。
- 背面尾巴保持白色與柔和輪廓，不加入粉色 X 或其他會讓嘴部設計語意混淆的記號。
- 四肢短小圓潤，姿勢重心穩定，整體比例偏可愛、幼圓、陪伴式。
- 耳朵、臉、頭部、身體與四肢的比例是角色 identity 的一部分，不是可任意變動的風格細節。新增圖必須接近核心設定圖的比例語言：頭部圓潤但不得膨脹成過大的球形頭，臉部留白與五官位置需維持安靜克制，身體不可被拉成過瘦、過長或擬人化的站姿，耳朵不可過長、過尖、過厚或變成垂耳造型。
- 生成或評估新圖時，不只檢查顏色、拉鍊嘴與道具，也必須檢查整體輪廓剪影與頭身比例是否仍讓人感覺是同一隻兔子；若只是具備相同要素但比例語言不同，應視為不合格並重生或重修。
- 同一組 runtime 兔子圖在透明畫布中的主體佔比必須一致。當多張圖以相同 `width`、`height` 與 CSS 比例放進網頁時，每隻兔子的視覺大小應相近；若生成結果的畫面佔比不同但角色風格、比例與語意已合格，應優先用後製調整透明畫布中的主體縮放、置中與底線位置，不應為了佔比問題重新生成。
- 主要道具可包含筆記本、便條、資料夾、鎖頭、警示牌、小卡片或其他中性溝通工具。

### 耳朵

- 正面耳朵為兩片直立、向外微張的長圓葉形。外側線條從頭頂兩側柔順上升，頂端完全圓，不尖；內側在頭頂中間留下淺 U 形間隙，不可完全貼合，也不可裂成過深 V 字。

## 生圖規則

後續 Agent 進行主角兔子生圖時，必須遵守：

1. 先讀取並檢視 `.agents/assets/characters/main_rabbit_role.png`。
2. 在 prompt 或 brief 中明確指出這張圖是角色設計核心 reference。
3. 描述時保留「純白、淡粉內耳與腮紅、柔和深灰線條、接近核心設定圖的頭身與耳臉身比例、圓潤幼圓比例、小眼睛、小型短橫向拉鍊嘴與其長度比例、溫柔陪伴感」等核心特徵。
4. 若使用影像生成工具，優先提供或引用核心設定圖作為 reference image，而不是只輸入文字 prompt。
5. 生成後檢查角色是否仍像同一隻兔子；若耳朵比例、頭身比例、臉部大小、身體大小、四肢比例、臉型、拉鍊嘴長度比例、表情、線條、色彩或氣質明顯偏離，應重試或修正 prompt。
6. 對外提供 prompt 時，同時附上產品語氣限制：私密筆記本、自我理解、界限核對、知情同意、溫柔陪伴，且不以教育服務自居。

## 新增前一致性檢視流程

每次生成、後製並準備把新的主角兔子圖片加入專案前，後續 Agent 必須先建立一次低成本的拼接閱覽圖，將新的候選圖與既有主角兔子圖放在同一張圖中比對。這個流程的目標不是製作正式交付素材，而是快速確認「看起來是否仍是同一隻兔子主角」。

流程要求：

1. **收集比對對象**：至少包含 `.agents/assets/characters/main_rabbit_role.png`、目前會進入 runtime 或正式文件流程的既有兔子圖，以及本次準備新增的候選圖。若本次新增多張圖，必須全部放入同一張拼接圖中。
2. **建立暫存拼接圖**：把所有比對圖以相同或接近的視覺高度排列成 contact sheet，可放在 `tmp/` 或其他不會進入正式資產流程的暫存位置。除非使用者明確要求保留，拼接圖不應被加入正式專案資產。
3. **使用一致背景檢查**：拼接圖應使用白底、淺灰底或透明棋盤格等能看清白色兔子輪廓的背景；若圖片是透明背景，應避免只在透明檢視器中判斷。
4. **先做整體觀感判定**：不要只逐項檢查顏色或拉鍊嘴；必須先縮小閱覽整張拼接圖，確認新圖在剪影、頭身比例、耳朵形狀、臉部留白、四肢短圓感、線條重量、淡粉使用與安靜陪伴氣質上，是否自然地屬於同一組角色。
5. **再做局部品質判定**：檢查拉鍊嘴長度、眼睛大小、腮紅濃淡、內耳形狀、道具遮擋、透明邊緣、主體佔比、水平中心與底部基線。若只是不一致的畫布佔比或站位問題，優先後製校正；若角色 identity 偏離，必須重生或重修。
6. **留下簡短判定**：在任務回報或相關工作紀錄中簡短說明已完成拼接閱覽，並指出結果是合格、需後製校正，或需重生。若沒有完成拼接閱覽，不得宣稱新兔子圖已準備好進入專案。

快速實作建議：

- 可用 `sharp`、ImageMagick、Pillow 或其他當下可用工具建立 contact sheet；不需要為這個檢查流程新增長期專案依賴。
- 拼接前可先把各圖正規化到接近相同高度與一致邊距，避免單純因畫布大小不同造成誤判。
- 若工具環境受限，至少應用現有圖片檢視工具開啟一張手動拼接或並排截圖；不可只憑單張新圖獨立判斷。
- 若新增圖會與特定既有圖片在同一 UI 流程切換，拼接圖應優先把這些圖排在相鄰位置，並額外檢查視覺大小與底部基線是否會在切換時跳動。

## 去背與透明背景品質規範

處理主角兔子圖片、UI mascot asset、宣傳素材或其他需要透明背景的圖像時，後續 Agent 必須遵守：

1. 去背後必須檢查主體邊緣，避免留下參差不齊的鋸齒、硬切邊、白邊、髒邊或低品質感。
2. 去背流程應對邊緣進行適度羽化、柔化或抗鋸齒處理，讓透明背景上的輪廓保持乾淨、柔和且自然。
3. 羽化不可過度，避免讓角色外輪廓變糊、變薄、失去原本柔和深灰線條或破壞白色主體形狀。
4. 若手中工具無法可靠進行邊緣柔和化處理，不要交付粗糙透明圖；應保留白底圖片，並告知使用者需改用專業圖片編輯軟體製作高品質去背版本。
5. 完成去背後，應在淺色、深色與透明棋盤格或等效背景上檢查邊緣品質；若邊緣缺陷明顯，應重新處理或回退為白底版本。

## 網頁載入用兔子圖片壓縮規範

處理會被 Vite 打包並在網頁 runtime 載入的主角兔子圖片時，後續 Agent 必須優先遵守本節，以維持目前已驗證的載入速度與視覺品質。

1. runtime 兔子圖片預設使用透明背景 WebP，不再使用大型 PNG 作為網頁載入格式，除非瀏覽器支援、透明度、色彩或邊緣品質有明確問題。
2. 目前本專案實測適合的 WebP 參數為 `quality: 90`、`alphaQuality: 100`、`effort: 6`。這組設定已將 `src/assets/story` 的四張兔子 PNG 從約 2.36 MB 降至約 200 KB，且保留透明邊緣、淡粉色細節與柔和深灰線條。
3. story flow 或 home screen 使用的單張兔子 runtime 圖，目標檔案大小應優先壓在 80 KB 以下；若畫面尺寸、細節或特殊姿勢使檔案較大，應先檢查是否超出實際顯示需求，並避免超過 120 KB。
4. runtime 兔子圖的解析度應依實際顯示尺寸保守設定。以目前 UI 而言，寬度約 1024px 已足以支援桌面最大顯示與高 DPR 螢幕；除非版面需求明確改變，不應輸出更大的 runtime 圖。
5. 壓縮後必須至少在深色背景上檢查透明輪廓、耳朵淡粉、腮紅、眼睛、嘴部與細線道具；若出現髒邊、色塊、鋸齒、輪廓破碎或角色氣質變硬，應提高品質或回退。
6. 同一組會在相同 UI 尺寸中切換或並列使用的 runtime 兔子圖，壓縮前應檢查 alpha bounding box，必要時用後製讓主體高度、水平中心與底部基線一致，避免同尺寸 `<img>` 中兔子忽大忽小。若只是畫面佔比不一致，不應直接重生已合格的角色圖。
7. 未被程式、文件或正式流程引用的兔子圖片應移除，避免累積過期資產；刪除前必須用全 repo 搜尋確認沒有引用。
8. `.agents/assets/characters/main_rabbit_role.png` 是角色設定 reference，不是網頁 runtime asset。除非使用者明確要求更新角色設定檔，不得為了網頁載入效能將該 reference 壓縮、改格式或替換。
9. 可用一次性工具產生 WebP，不需要把圖片壓縮 CLI 加入專案依賴。已驗證可用的命令範例：

```powershell
npx sharp-cli --input "src/assets/story/*.png" --output tmp/webp-q90 --format webp --quality 90 --alphaQuality 100 --effort 6
```

## 禁止方向

主角兔子不得被生成或描述為：

- 露骨、煽情、NSFW、情色化或高刺激慾望的角色。
- 穿著或擺出明確 BDSM 實踐、性暗示、羞辱、施壓或支配他人的姿態。
- 鼓勵使用者嘗試特定 BDSM 項目、挑戰高風險項目或忽視同意。
- 用尖銳、恐怖、獵奇、陰暗、壓迫或過度成人化的視覺語言呈現。
- 被改造成固定人格、屬性、角色標籤或測驗結果分類的象徵。

若必須處理 BDSM、limits、風險警示或分享語意，兔子只能以陪伴、提醒、整理、核對與保護界線的方式出現。

## 建議 prompt 結構

撰寫主角兔子 prompt 時，建議包含：

1. `reference`：明確要求參考 `.agents/assets/characters/main_rabbit_role.png` 的同一角色設計。
2. `character`：純白圓潤插畫兔子、淡粉內耳與腮紅、柔和深灰線條、小眼睛、小型短橫向拉鍊嘴；耳朵、臉、頭部、身體、四肢與拉鍊嘴長度比例必須接近核心設定圖；尾巴保持白色，不加 X 記號。
3. `role`：溫柔、安靜、陪伴使用者整理秘密檔案與核對界線。
4. `pose`：具體姿勢或道具，例如拿筆記本、坐著思考、抱著資料夾、指向提醒卡。
5. `mood`：安心、溫和、克制、非刺激性。
6. `constraints`：非 NSFW、非露骨、非情色化、不呈現 BDSM 操作教學或鼓勵行為。
