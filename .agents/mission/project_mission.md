# Boundary Notes 專案使命

## 文件角色

本文件是 Boundary Notes 的產品目的、使用者價值、非目標與知情同意原則的 canonical owner。它不保存 route、CSS、schema、deployment 或單一頁面的 current implementation。

## 名稱與品牌

產品名稱公式為「XX的祕密檔案」：

- 未輸入暱稱時顯示「兔子的祕密檔案」。
- 輸入暱稱後顯示「{暱稱}的祕密檔案」。
- `XX` 是動態命名公式，不是等待補完的 placeholder。

穩定品牌名稱為 **Boundary Notes**，正式網域為 `https://boundarynotes.com`。Boundary Notes 是品牌與網域層，不取代各語系與使用者名稱組成的動態產品標題。

公開品牌、SEO 與分享 metadata 必須保護使用者敏感資訊，不包含暱稱、答案、檔案 ID 或 query。品牌選擇的長期取捨記錄於 `.agents/decisions/decision_history.md`；current SEO implementation 由 `.agents/skills/professional/current_system_state.md` 路由到 code。

## 核心定位

Boundary Notes 是一本協助自我理解，以及協助人與人之間核對 BDSM 互動項目界限的私密筆記本。

內容可以具有教育性質，但產品不以教育為主要目的，也不以教育者、教育工具或教育服務對外宣稱。對外描述優先使用「筆記本」；只有描述一般功能、風險、操作元件或開發工具時才使用「工具」。

產品的核心理念是知情同意。使用者應能理解自己填寫了什麼、分享了什麼、對方會看到什麼，以及這些資訊只能支持事前溝通與界限核對。

## 核心目標

- 協助使用者記錄自己對具體 BDSM 互動項目的經驗、興趣、界限、條件與備註。
- 讓使用者用自己的步調理解目前感受，不把不確定、空白或改變視為錯誤。
- 讓使用者能檢閱、保存、比較與分享自己的秘密檔案。
- 讓第三方在閱讀分享檔案時理解它是溝通起點，而不是同意或承諾。
- 讓界限、興趣與狀態能以新版本呈現隨時間改變的事實。

題庫以「互動項目」為導向，不以人格、身份、角色、屬性或取向標籤替使用者分類。互動方向、強度、時間與情境條件應能被分開理解；正式資料契約由 `.agents/specs/question_bank_and_secret_file_system.md` 管理。

## 核心能力

- 建立、作答與檢閱自己的秘密檔案。
- 在本機保存、更新、刪除與匯入／匯出秘密檔案。
- 建立唯讀雲端版本並以專屬連結分享。
- 閱讀他人的唯讀秘密檔案。
- 以新版本更新既有內容，不覆寫已分享版本。
- 比較多份秘密檔案，觀察狀態與界限變化。
- 支援繁體中文、簡體中文、日文與英文，以及手機和桌面體驗。

具體流程與 layout 由 `.agents/specs/product_experience.md` 管理；current routes 與 implementation 由 `.agents/skills/professional/current_system_state.md` 管理。

## 產品非目標

Boundary Notes 不做：

- 社交平台、配對、約會、尋找對象或社群。
- 鼓勵、推廣、推薦、排名或挑戰任何 BDSM 行為。
- 根據答案替使用者貼上人格、角色、屬性或取向標籤。
- 以檔案取代現實中的事前溝通、能力確認、風險討論與當下同意。
- 將分享連結設計成壓力、承諾、契約或要求他人同意的工具。
- 使用露骨、挑逗、獵奇、高刺激慾望或 NSFW 的方式呈現內容。
- 以教育工具、教育服務或教育者身分對外宣稱。

## 知情同意 Invariants

- 經驗、興趣、可接受程度、界限與備註是不同語意，不可合成一個「同意分數」。
- 空白、低分、不確定、矛盾、拒絕或附帶條件都不得被 UI 自動解讀為可接受。
- 檔案只代表填寫當下的自我描述；現實互動中的同意必須能持續確認與撤回。
- 分享前必須讓使用者理解內容、可見範圍、完整連結可被再次轉傳，以及雲端保存邊界。
- 雲端檔案建立後長期不可由一般使用者修改、刪除或撤回且不自動過期；只有網站管理員因法律要求處理。產品不得暗示使用者之後可以撤回已建立資料。
- 使用者自主決定分享位置。產品可以揭露風險，但不替使用者宣稱哪些對象、平台或空間可信。
- 閱讀他人檔案時必須提醒：檔案不代表永久、即時或不可撤回的同意。
- 高風險或可能觸及法律、醫療、身體傷害、心理傷害或第三方未同意暴露的項目必須提供清楚警示。

## 風格與角色

產品語氣溫和、溫柔、有溫度，保留私密筆記本感，避免冷冰冰的醫療表格或過度工程化語氣。文案不羞辱、不獵奇化、不道德評判，也不暗示使用者「應該嘗試」。

守密兔是安靜、可信任、陪伴使用者整理與核對界限的角色。牠不是 BDSM 實踐者、誘惑角色、測驗分類或教育者。敏感情境使用中性道具、表情與空間隱喻，不直接呈現行為。

角色 identity、拉鍊嘴、比例、色彩、姿勢、去背與 runtime asset 規格由 `.agents/skills/visual/main_rabbit_image_generation.md` 及 `.agents/assets/characters/main_rabbit_role.png` 管理。

## 高風險內容

高風險警示的目的只在辨識風險並促使現實溝通，不提供操作教學、規避法律的方法或安全保證。涉及具體法律、醫療或新高風險判斷時，必須另外查證可靠且符合目標地區的來源。

BDSM 領域語意、limits、safeword、分享與警示原則由 `.agents/skills/domain/bdsm_consent_background.md` 管理。
