# 專案使命與產品方向

## 文件目的

本文件定義 `bdsm_boundary_test` 的長期產品目標、核心邊界、語氣方向與後續 Agent 應遵守的產品判斷基準。後續 Agent 在規劃、實作、撰寫題庫、調整 UI/UX、建立資料結構或修改使用者可見文案前，必須先讀取本文件。

## 專案名稱

產品名稱為「XX的祕密檔案」。

`XX` 會根據使用者輸入的暱稱改變。使用者尚未填入暱稱時，`XX` 與完整產品名稱都應預設為「兔子的祕密檔案」；使用者填入暱稱後，才顯示為「{暱稱}的祕密檔案」。

若未來需要建立品牌、URL、metadata、分享標題或匯出檔名，應保留這個動態命名特性，不要把 `XX` 固定成特定人名、角色名或專案代號。任何未命名狀態、空狀態、預覽狀態或匿名範例，都應優先使用「兔子的祕密檔案」作為預設顯示。

穩定品牌名稱為 **Boundary Notes**，正式網域為 `https://boundarynotes.com`，採用無連字號的 `boundarynotes`。網域已購買；production Hosting 驗證後綁定 apex domain，並讓 `www.boundarynotes.com` redirect 到 apex domain。Boundary Notes 是品牌與網域層，不取代 app 內各語系的產品標題；產品標題仍依使用者名稱與語系動態顯示，例如英文預設為「Bunny's Secret File」、命名後為「{暱稱}'s Secret File」。公開可索引頁的 SEO 標題與描述應以界線整理、知情同意、界線溝通筆記本及 Boundary Notes 品牌為主，不得把網站宣稱為教育工具、教育服務或具有教育目的；秘密檔案與功能 URL 的社群 metadata 一律使用不含暱稱、作答、檔案 ID 或 query 的通用 Boundary Notes 文案與圖片。

## 核心定位

本專案是一本協助自我理解，以及協助人與人之間核對界限的筆記本。內容與呈現可以具有教育性質，但網站不以教育為主要目的，也不以教育者、教育工具或教育服務的身分對外宣稱。對外描述產品時，應優先使用「筆記本」而不是冷冰冰的「工具」；只有在描述一般功能、風險、操作元件或開發工具時才保留「工具」。專案內容會包含大量 BDSM（愉虐）相關互動項目，但產品目的不是刺激慾望、鼓勵實踐、撮合關係或推薦使用者嘗試特定項目。

產品應最大程度強調「知情同意」，並在自身實作中把知情同意作為核心理念實踐：使用者應能清楚理解自己填寫了什麼、分享了什麼、對方會看到什麼，以及如何用這些資訊支持事前溝通與界限核對。

## 核心目標

使用者可以透過測試理解自己對各 BDSM 互動項目的喜好、經驗狀態與接受程度，並透過產生專屬連結與對象溝通自己的界限。

題目應以「互動項目」為導向，而不是以「人格屬性」或「身份標籤」為導向。本專案要測量使用者對具體項目的傾向，不應把使用者歸類成固定角色、族群、取向或人格標籤。

典型題目結構應同時支援分類層與細項層。正式題庫、回答 JSON、本地保存、雲端分享與 Google Sheet 更新流程定義於 `.agents/specs/question_bank_and_secret_file_system.md`。

典型分類題目：

- 分類標題：例如「拍打類項目(主導側)」
- 分類描述：例如「對對方進行包含打屁股在內的拍打相關衝擊遊戲」
- 使用者回應：經驗程度、喜好程度、備註

典型細項題目：

- 項目描述：例如「施予臀部拍打-重度」
- 使用者回應：經驗程度、喜好程度、備註

題庫設計應將互動方向、強度、時間長度與情境條件拆成可辨識的不同問題。例如「施予臀部拍打-重度」與「接受臀部拍打-輕度」是不同題目；主導方與接受方也應視為不同互動方向。

## 核心功能

- 繁體中文、簡體中文、日文、英文四語公開搜尋入口必須直接沿用既有前導畫面，不得為 SEO 新增或插入獨立 landing。首次使用者由 `/{locale}` 開始既有前導；已完成前導命名的使用者由相同入口直接前往同語系 Home。顯式重播前導仍可直接開啟 `/{locale}/intro`，而已播放過的前導應提供低視覺重量的文字底線「跳過」連結。建立新檔案的重播劇情必須先選擇問題範圍才可跳過；時光機則必須先選擇兩份檔案才可直接略過敘事進入比對。公開 metadata 可自然使用 BDSM 界線、知情同意、界線整理與界線溝通等筆記本語意，但不得把網站描述為教育工具、教育服務或具有教育目的，也不得使用契合度、配對或角色測驗語意。
- 主頁應作為多個獨立功能頁面的入口，而不是永遠停留在單一 landing 或單一流程。目前主頁入口基準為「創建新檔案」、「查看舊檔案」、「搭乘時光機」、「關於」、「設定」與「贊助本網頁」；其中「關於」、「設定」與「贊助本網頁」應使用較低視覺重量的文字連結入口，不與主要功能使用同等重量的大按鈕，也不以右上角 icon 破壞畫面重心。「贊助本網頁」同時保留於祕密檔案檢視頁，兩處開啟同一個贊助小視窗。實作時應讓入口能逐步擴充，不把前導劇情、主頁與後續功能綁成同一個大型畫面。
- 「關於這隻兔子」應以接近一屏完成為優先，把專案使命、守密兔的陪伴角色與核心界線合併成一段清楚敘述，不得再用多組卡片、原則清單或能做／不能做清單重複同一件事，也不得展示實作細節或內部防護機制。畫面只另行保留敏感內容提醒、GitHub Issue、電子郵件、使用條款、隱私權政策與版本歷史等必要入口；前導劇情重播疊放於兔子插畫區，返回入口置於頁面上方角落。法律頁以新分頁開啟、使用單頁靜態閱讀版面且不設返回鍵；版本歷史以獨立頁面依版本列出正式釋出內容。
- 讓使用者接受測試，建立自己的秘密檔案。
- 讓使用者檢閱自己作答完成的祕密檔案。
- 建立該秘密檔案的專屬連結，供使用者傳遞給第三方分享。
- 讓使用者檢閱他人專屬連結中的秘密檔案內容。
- 更新或編輯特定秘密檔案；實作上應以新建資料表示新的版本，不直接覆寫舊資料。
- 載入多份秘密檔案，透過時間軸觀察同一使用者或同一主體的變化。
- 查看本地持久儲存的歷史秘密檔案。

## 輔助功能

- 支援手機與電腦版面。這不是單純縮放桌面版，而是應依螢幕尺寸提供不同資訊密度、導覽方式與操作布局。
- 支援繁體中文、簡體中文、日文、英文四種語系，以獨立語系 URL 呈現；URL 語系優先於已保存語系，無法辨識來源語系時以英文作為公開文案與圖片 fallback。
- 僅需單一佈景風格，不需要明亮/黑暗模式切換。

## 產品邊界與非目標

本專案不包含下列方向：

- 社交平台、配對、約會、尋找對象或社群功能。
- 鼓勵、推廣、推薦或排名任何 BDSM 行為。
- 依測驗結果替使用者貼上固定人格、角色、屬性或取向標籤。
- 讓使用者用他人檔案替代現實中的事前溝通、確認同意或風險討論。
- 把分享連結設計成壓力工具、承諾工具、契約工具或要求對方同意的工具。
- 以露骨、挑逗、高刺激慾望的方式呈現題目、說明或插畫。

## 知情同意實作原則

產品設計必須讓同意、界限與風險意識具體落在流程中：

- 題目與結果要清楚區分「經驗」、「感興趣」、「可接受」、「備註」等不同語意，不可用單一分數暗示同意。
- 使用者沒有填答、低分、備註中拒絕或表達不確定時，不得被 UI 解讀成可接受。
- 分享前必須讓使用者知道分享內容、可見範圍與可能風險。
- 建立雲端分享連結前應再次說明持有完整連結者可讀、雲端檔案長期保存且一般使用者目前無法自行修改或刪除；使用者按下明確的建立按鈕，即表示已閱讀使用條款與隱私權政策並同意上傳該份可能含有敏感內容的檔案，不另加 checkbox。
- 檔案連結應被視為敏感資料入口；雲端檔案一經建立即長期不可編輯、不可由一般使用者刪除或撤回且永不過期，僅能因法律要求由網站管理員移除。分享前必須清楚揭露這項保存邊界，不能暗示之後可以撤回連結。
- 分享位置由使用者自主決定；產品可提醒完整連結持有者可讀且連結可能被再次轉傳，但不得宣稱不得公開張貼，或替使用者決定哪些對象、平台或空間可信。
- 檢閱他人檔案時，UI 應提醒此檔案只支援溝通，不能代表永久、即時或不可撤回的同意。
- 編輯舊檔案時以新版本表示變化，讓使用者能看見界限、興趣與狀態會隨時間改變。
- 高風險或可能觸及法律風險的項目，必須在題目或檢閱階段提供清楚警示。

## 風格與語氣

整體風格應以溫和、私密的筆記本感為主，而非情色展示或刺激慾望；內容可以具有教育性質，但不應讓產品以教育者自居。語氣應溫和、溫柔、有溫度，避免冷冰冰的醫療表格感或過度工程化的產品感。

產品有一隻插畫兔子作為主角，與使用者互動，讓整體體驗可愛、親近、陪伴感較強。兔子插畫不得過度露骨，不應呈現 NSFW 畫面；可用隱晦、象徵、日常小物或情緒表情來表達情境。兔子的嘴巴使用小型拉鍊嘴，代表守信、保密與隱私，不使用 X 形嘴；背面尾巴保持白色，不加入粉色 X 記號。

主角兔子的核心人物設定檔為 `.agents/assets/characters/main_rabbit_role.png`。後續生成、修改或延伸主角兔子圖片時，必須遵守 `.agents/skills/visual/main_rabbit_image_generation.md`，並以該設定圖作為同一角色的最高優先級視覺 reference。

文案應避免羞辱、獵奇化、道德評判或暗示「應該嘗試」。比較好的方向是提醒、陪伴、核對與留白，例如「先記下目前的感覺也可以」、「不確定本身就是很重要的答案」、「這份檔案是溝通起點，不是同意書」。

## 高風險項目提示

題庫若包含可能觸發法律、醫療、身體傷害、第三方非自願暴露或重大心理風險的項目，必須特別標示警示。例子包含但不限於：

- 室外露出、公共場所、可能讓未同意第三方被迫看見或參與的情境。
- 見血、穿刺、窒息、束縛、限制行動、藥物或酒精影響判斷的情境。
- 任何可能造成持續傷害、留下痕跡、感染風險、意識改變或難以即時停止的行為。

警示文案應以風險辨識與知情同意為核心，不應提供操作教學、規避法律的建議或降低風險的虛假保證。
